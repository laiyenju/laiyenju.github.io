> 從政治學系到新聞媒體，再回到學校涉入 UX 研究領域（還一股熱血地卡在自然語言處理的研究論文中），我真的很難跟爸媽說清楚自己在幹嘛。

在台灣的許多關於設計社團內，以前常見到的 UX／UI 職位名稱，已逐漸 UX、UI 分家，對許多設計師來說是好消息，對 UX 研究員而言更是。但專精化的職位劃分只是公司分工的權責與範圍界定被細分，個人能更深入所負責的事項，卻不代表設計師或研究員的養成過程中，學習的項目也能跟著減少，我真覺得未來公司會更傾向每個部門都設置專屬的 UX 工作者（當然前提是有足夠的財力），否則 Google 就不會開出 UX Engineer 或 UX＋AI 的職缺。

即使如此，還是會很常聽到許多人踩上愚昧之丘大喊著「使用者體驗」、「使用者為中心」，做個問卷調查、訪個談就完成所謂的 UX。雖然每個人都可以是設計師或研究員，但這樣真的有點太隨便、有點可惜。讓許多人對 UX 有這麼深的誤會，或許某種程度上 UX 要負點責任，如果 UX 能被我們更白話地介紹，或許上面的 meme 能成為過去式。

- Persona、mental model、service design⋯⋯是什麼？
- UX、UI、Information Architecture⋯⋯又是什麼？

第一項的各種設計研究名詞，以及第二項的各種設計相關職稱，讓大家能望文生義，但要詳細說明又詞不達意，Josh Munn 寫的 Why Design-isms are a problem 文章點出了這類設計主義積病，所以他又製作海報示範如何白話描述這些項目。

## 不掛 UX 職位也能做 UX

看過經白話翻譯的描述後，會發現能跟 UX 搭上邊的項目好多，大家都在玩 UX 連連看，很多東西只要跟人有關，就能 X 上去，成為 UX、CX，老師之前上課還介紹了 Kansei Engineer（感性工學）。這這裡大概能了解 UX 的「最基本能力」就是要在理性與感性間轉換，因此要做好 UX 工作除了觀察外，還要有：

- 同理心
- 數據分析能力
- 將數據、資料轉換為資訊與知識的能力
- 表達能力

既然是能力，那就能夠培養。之前曾看到別人的推文表示：跟 UX 職同事共事是個良好的使用者體驗。是啊，只要是人，就都是使用者，從工作方式就能開始訓練自己；數據分析跟轉換資訊的能力就不用多說了，這是基本功中的基本功；當該做的研究都做了，要如何說得流暢、說得白話、說得讓人躍躍欲試自己的解決方案？對自己的解決方案有信心，就要臉不紅氣不喘地運用表達能力將自己的解決方案推銷出去。但這對我這種生性內向的人來說，表達能力是個大門檻，必須時常強迫自己做訓練，而我近期發現 Twitch 是個好東西。

### 在 Twitch 自言自語

我看到馬世芳在 Twitch 開設一個專門播音樂的廣播頻道後，也跟著觀察了 Twitch 平台的生態後覺得很適合內向者練習口語表達能力，因為：

1. Twitch 主要直播遊戲，所以直播遊戲以外的節目不一定會吸引到人觀看
2. 用 OBS 就能串流電腦畫面到自己的 Twitch 頻道上，可以不用露臉

內向的人就是不喜歡露臉、也不想要被注目，但又想偶爾獲得關注與回應啊～Twitch 的生態正合適，所以我就在 Twitch 直播自己寫程式，邊寫邊說自己的解題想法與學習目標，之後也計畫用來練習小組報告，解決學校遠端教學的作業。

總之，當人人都可以是設計師與研究員時，有限的 UX 職位已經不是重點，將 UX 作為一種生活態度與工作態度，也很不錯。
